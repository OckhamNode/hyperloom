---

**Hyperloom** is a ridiculously fast, fine-grained thread-safe Memory Graph and Context Broker. It allows 10,000+ AI Agents (like Claude, GPT-4, or Rovo) to simultaneously read and write contextual data to a shared global memory state in real time without blocking.

It bypasses bulky JSON locking and implements a massive **Concurrent Trie Forest** powered by `sync.RWMutex` locks at the node level. Features an automatic **Rollback Engine** that prunes hallucinated or errored data.

## 🚀 Features

- **Blazing Fast In-Memory Channel Logs**: Stream 100k+ events/sec before blocking.
- **Node-Level Locking**: Agents reading `/agent_1/memory` do not block agents writing to `/agent_2/memory`.
- **Smart Append Merging**: Automatically deep-merges JSON Objects and appends elements to JSON Arrays.
- **Transaction Rollbacks**: Any agent that halts or breaks mid-transaction automatically has its "shadow branch" wiped from the global Graph.
- **Real-Time Websocket Pub/Sub**: Fans out localized tree updates instantly.

## 🛠️ Installation

Simply clone and run:

```bash
git clone https://github.com/OckhamNode/hyperloom.git
cd hyperloom
go build -o hyperloom .
./hyperloom
```
The central broker spins up on `ws://localhost:8080` globally.

## 🔌 MCP Bridge for Claude Desktop

Hyperloom ships with a native **Model Context Protocol (MCP)** server built securely into the Go framework! This means Claude can natively read and write to your highly concurrent graph immediately without you writing code.

Simply add this to your `claude_desktop_config.json`:

```json
{
  "mcpServers": {
    "hyperloom": {
      "command": "go",
      "args": ["run", "./cmd/mcp/main.go"]
    }
  }
}
```

Claude will automatically gain two new tools: `read_global_memory` and `write_global_memory`. Any time Claude updates the graph, other subscribed agents will be alerted!

## 🌎 Real-World Integrations

Hyperloom is designed to unify multiple AI models into a shared namespace. Below are examples of how LLM's connect to the matrix.

### Example: Connect Anthropics Claude (Python)

If Claude is running a complex long-term task, it can stream context patches to Hyperloom. If Claude hallucinates or crashes, we simply call `revert()`.

```python
import websocket
import json
import requests

# 1. Connect to Ingest Socket
ws = websocket.WebSocket()
ws.connect("ws://localhost:8080/ingest")

tx_id = "claude_tx_901"

# 2. Stream Context as Claude thinks
diff = {
    "agent_id": "claude-3-opus",
    "tx_id": tx_id,
    "path": "/global/project_alpha/summary",
    "op": "APPEND",
    "value": ["Analyzed the database architecture."]
}
ws.send(json.dumps(diff))

# 3. Commit state to Graph when finished
requests.get(f"http://localhost:8080/commit?tx_id={tx_id}")
```

### Example: Syncing Rovo Agent State (NodeJS)

A Rovo Agent can subscribe to changes generated by Claude and alter its behavior in real time!

```javascript
const WebSocket = require('ws');

// Listen intensely only to Project Alpha's folder
const ws = new WebSocket('ws://localhost:8080/subscribe?path=/global/project_alpha');

ws.on('message', function message(data) {
  const update = JSON.parse(data);
  console.log(`[Rovo HUD] Alert in ${update.path}! New value:`, update.value);
  // Rovo immediately reacts to Claude's database summary!
});
```

### Example: GPT-4 Hallucination Rollback

If GPT-4 generates corrupted JSON, Hyperloom cleans it up safely.

```python
# GPT-4 stages catastrophic data...
ws.send(json.dumps({
    "agent_id": "gpt-4",
    "tx_id": "gpt_tx_bad",
    "path": "/global/project_alpha/security",
    "op": "SET",
    "value": "DROP TABLE USERS;" 
}))

# System throws validation error!
# REVERT instantly clears the "gpt_tx_bad" shadow memory globally.
requests.get("http://localhost:8080/revert?tx_id=gpt_tx_bad")
```
